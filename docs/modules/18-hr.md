# ระบบที่ 17: พนักงาน (HR) — เวลาเข้างาน ขาด ลา มาสาย

> ออกแบบโดย Fable 5 (2026-07-12) ตามคำขอเจ้าของ: "ระบบ HR พนักงาน ขาด ลา มาสาย"
> ไฟล์หมายเลข 18 — ทะเบียนระบบ = **ระบบที่ 17** · AppSystem type `HR` (feature) สร้างหลายชุด/เชื่อมระบบ business ได้ตามสถาปัตยกรรมเดิม

## 1. ขอบเขต v1 (Time & Leave — ตามที่เจ้าของขอ)
- **ทะเบียนพนักงาน**: ชื่อ/ตำแหน่ง/เบอร์/วันเริ่มงาน/ค่าแรง (รายเดือน|รายวัน|รายชั่วโมง — เก็บไว้ใช้ payroll P2) · ผูกกับ `User` (login) ได้ optional และผูก `BookingStaff`/พนักงานร้านอาหาร ได้ (คนเดียวกัน)
- **กะ/ตารางเวลา**: กำหนดกะ (เช้า/บ่าย/custom เวลาเข้า-ออก) + จัดตารางรายสัปดาห์ต่อคน + วันหยุดประจำสัปดาห์
- **ลงเวลา (Clock in/out)**: พนักงานกดผ่านมือถือ (login) หรือ**หน้าจอ kiosk ของร้าน + PIN 4 หลักต่อคน** (ร้านเล็กสะดวกสุด) · เก็บเวลา+อุปกรณ์ · GPS/รูปถ่าย = 🔜
- **สถานะอัตโนมัติจากกะ**: มาปกติ / **มาสาย** (เกิน grace period ที่ตั้ง เช่น 15 นาที) / **ขาด** (ไม่ลงเวลาในกะ) / ลา / หยุดตามตาราง — สรุปเป็นปฏิทินรายเดือนต่อคน
- **การลา**: ประเภท (ลาป่วย/ลากิจ/พักร้อน + custom) + โควตาต่อปีต่อคน → พนักงานยื่นในมือถือ → **หัวหน้าอนุมัติ/ปฏิเสธ** (แจ้งเตือน) → ตัดโควตา แสดงยอดคงเหลือ
- **OT**: บันทึกชั่วโมง OT (ยื่น+อนุมัติ) — คิดเงิน P2
- **รายงาน**: สรุปเดือนต่อคน (วันทำงาน/สาย กี่ครั้ง-กี่นาที/ขาด/ลาแยกประเภท/OT) + export CSV — พร้อมเป็น input ทำเงินเดือน

## 2. Data model (systemId-scoped ตามแบบแผนเดิม)
`HrEmployee` (userId?, name, position, pin, wageType, wageSatang, startedAt, active) · `HrShift` (name, startMin, endMin, graceMin) · `HrSchedule` (employeeId, weekday|date, shiftId | OFF) · `HrTimeEntry` (employeeId, clockInAt, clockOutAt?, source KIOSK|MOBILE, status OK|LATE(lateMin)|ABSENT-generated) · `HrLeaveType` (name, quotaDaysPerYear) · `HrLeaveRequest` (employeeId, typeId, dateFrom/To, halfDay?, reason, status PENDING|APPROVED|REJECTED, approvedBy) · `HrOtEntry` — เงินสตางค์/กติกา scope เหมือนโมดูลอื่น

## 3. การเชื่อม (จุดแข็งของ SHARK)
- **Booking**: HrEmployee ↔ BookingStaff คนเดียวกัน → ลา/ขาดแล้ว **slot จองของช่างคนนั้นปิดอัตโนมัติ** (killer feature ที่ FlowHR เดี่ยวๆ ทำไม่ได้)
- **Account (P2 payroll)**: สรุปเดือน → คิดเงินเดือน/หักสาย-ขาด/OT → สร้างรายการค่าใช้จ่ายเงินเดือน + WHT ภงด.1 ลงบัญชีอัตโนมัติ
- **Meeting**: แจ้งเตือนคำขอลา/อนุมัติเข้า channel · **AI พนักงาน**: "เดือนนี้ใครสายบ่อยสุด" / Daily Brief รวมคนลาวันนี้
- **Restaurant/Hotel**: staff ผูกกะเดียวกัน

## 4. Phasing
**P1**: ทะเบียน+กะ+ตาราง+clock in/out (mobile+kiosk PIN)+สาย/ขาดอัตโนมัติ+ลา (ยื่น-อนุมัติ-โควตา)+รายงานเดือน+เชื่อม BookingStaff · **P2**: payroll (สลิปเงินเดือน+ลงบัญชี+ภงด.1)+OT คิดเงิน+ประกันสังคม · 🔜: GPS/selfie clock-in, ประเมินผลงาน, ปฏิทินวันหยุดนักขัตฤกษ์ไทย preset
